# Generic AI Adapter

> 도구별 어댑터가 없는 AI(웹 UI / 단순 채팅 / 로컬 LLM 등)를 위한 fallback.
> 본 방법론의 *제0원칙(이식성)* 의 최후 안전망.

## 부팅 절차 (사람이 수동으로)

1. 저장소 루트에서 `.ai/context.json`을 열어 *전문을 그대로* AI 채팅창에 붙여넣기.
2. `must_read` 배열의 파일들을 **순서대로** 열어 차례로 붙여넣기.
3. 작업 성격에 따라 `must_read_optional`에서 골라 붙여넣기.
4. AI에게 첫 지시: "위 컨텍스트만으로 `.ai/checkpoint.md`의 §'다음 사람에게' 첫 항목을 수행해줘."

## 세션 종료 절차 (수동)

AI에게 다음과 같이 요청:

> ".ai/checkpoint.md를 현재 파일의 필수 섹션 형식으로 갱신해줘. 과장·칭찬·추측 없이, 다음 AI가 이어받는 데 필요한 사실과 첫 행동만 남겨줘."

응답을 `.ai/checkpoint.md`에 저장(덮어쓰기).
이전 checkpoint를 보존하려면 `40_dev/snapshots/checkpoints/YYYY-MM-DD_HHMM.md`로 복사 후 갱신.

## 한계 (이 어댑터의 정직 모드)

- **자동 hook 없음** → 사용자가 매 세션 종료 시 수동 갱신. 누락 시 시스템 단절.
- **트랜스크립트 분석 불가** → L3 보너스 신호 없음. 주 신호원은 Git/PR + L1 관찰 로그.
- **부팅 시간 길어짐** → 자동 컨텍스트 주입이 없어 사람이 5~10개 파일을 손으로 붙여넣어야 함.

## 권장 사용처

- 인터넷 없는 로컬 LLM 환경 (작업 인계만 보장하면 충분)
- 처음 보는 AI 도구로 *최소 검증*만 할 때
- 도구별 어댑터를 작성하기 전 임시 사용
