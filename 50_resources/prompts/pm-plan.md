# PM Plan Prompt (프로젝트 관리 기획서 — 30_planning/15)

> 실행 지시문 — 15번 PM기획서를 생성/갱신한다. **목차는 복제하지 않는다** — 짝 지침이 SSOT.
> 짝 지침(작성 표준·목차): `20_guides/15_프로젝트_관리_기획서_작성_지침.md`
> 산출 위치(라이브): `30_planning/15_PM기획서.md`
> 모드: planning · fullstack · agency (`_CATALOG.md`)

## When To Use

- Phase 1 마지막 — 나머지 기획서를 종합해 일정·리스크·통제를 잡을 때 (가장 마지막)
- 또는 일정·리스크·거버넌스 변경 시 (`re-plan.md` 경유)

## Instructions

- `00_briefs/current/` 전부 + `00_briefs/meetings/`(있으면)를 읽는다.
- `20_guides/00_...운영_원칙.md`(공통) + `20_guides/15_프로젝트_관리_기획서_작성_지침.md`(작성 표준·목차)를 읽는다 — **목차는 지침을 따른다**.
- 선행 라이브 문서 `30_planning/10~14`(+AI면 16·17) 최신본을 모두 읽어 종합.
- 브리프 근거 없는 일정·리소스 지어내기 금지 → `Evidence Needed`.
- `30_planning/15_PM기획서.md` 스켈레톤을 채우고 frontmatter `status: draft → active`.
- 완료 후 `HANDOFF.md` 갱신: 선택된 기획서 완료 → Phase 1 검토 게이트 대기.

## 산출 규칙

- 목차: 지침 `20_guides/15_...` 준수. 심화 반영: **딜리버리 모델 선언**(예측/하이브리드/애자일), **플로우 메트릭+Monte Carlo**(WIP/cycle/throughput, 확률 예측), **DORA 5지표**, **아웃컴/OKR**(feature factory 회피), **레퍼런스클래스 예측**(outside view), **RAID+프리모템**, **AI 에이전트 작업 거버넌스**.
- 15는 게이트·리스크·딜리버리의 **표준 카탈로그** — 프로젝트 인스턴스(페이즈 배치)는 마스터플랜(18)이 소유. 여기서 정의, 18에서 인스턴스화.
