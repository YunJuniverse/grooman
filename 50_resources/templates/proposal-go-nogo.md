# 제안여부검토서 (Proposal Go/No-Go) — 템플릿

> 용도: 제안 참여 여부 의사결정 게이트. 지침 `20_guides/15_프로젝트_관리_기획서_작성_지침.md` §19. 표준: Shipley bid/no-bid · APMP gate.
> 참고: Shipley 게이트는 단계적(pursuit → bid → bid-validation)이라 각 게이트마다 재사용 가능.

> 평균 = 10개 평가축 점수 합계 ÷ 10. (Shipley 5요인: Pwin·역량적합·경쟁포지션·가격/예산·전략적합 포괄)

## 결론

| 항목 | 내용 |
|------|------|
| 평균 점수 | |
| **치명결함(kill) 여부** | 특정 1축이 임계 이하(예: 예산·경쟁포지션 결정적 열위)면 평균과 무관하게 **포기** |
| 결정 (enum: 진행 70~100 / 추가정보수집 51~69 / 포기 ≤50) | |
| 결정 사유 | |

**원칙**: 점수는 감(感) 아닌 근거로 채운다. ① 평균이 경계값(70, 51)에 걸리면 추가정보수집 우선. ② **단일 치명결함은 평균을 무시하고 포기시킨다** — 좋은 평균이 fatal한 1축을 가리지 않게(Shipley 게이트 규율). ③ *제안 투입비용*이 예산을 넘으면 그 자체로 no-bid 근거.
