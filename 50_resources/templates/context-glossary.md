# 도메인 용어집 (Context Glossary) — 템플릿

> 용도: 팀(과 AI)이 같은 단어를 같은 뜻으로 쓰기 위한 **용어집만** 담는다(스펙 아님 — 그건 `prd.md`·`architecture.md`). **경량 문서 모드**의 3번째 축. SI 단계별 표준용어는 `glossary.md`(별개) — 이 문서는 *도메인 개념 canon*. 지침 `20_guides/11_서비스기획서_작성_지침.md` §19 참조.
> **위상 = 유비쿼터스 언어(DDD)**: 여기 표준어는 대화·문서·UI·테스트·**코드에 그대로** 나타나야 한다(사전이 아니라 *언어 계약*). 용어 드리프트는 속도·품질을 갉는다.
> **SKOS 매핑**: 표준어 = prefLabel · 허용 동의어 = altLabel · `_Avoid_` = hiddenLabel(검색엔 잡히되 표시 금지).

## 용어

> 최소형 = 표준어 + 정의 + `_Avoid_`. 아래 메타는 *용어당 선택*(소규모는 생략, 도메인 커지면 채움).

**<표준 용어 A>**  `[context: <바운디드 컨텍스트, 예: Sales>]` · Status: Approved · Owner: @name
<정의 1~2문장. 무엇에 귀속되고 무엇을 하는지.>
_Avoid_: <혼동어1>, <혼동어2> (← 왜 피하는지)
Code/UI: `OrderLine` / "주문 항목"  ·  See also: <관련어1>, <관련어2>

**<표준 용어 B>**:
<정의>
_Avoid_: <혼동어>

> **Status** enum: Draft / Approved / Deprecated. **Owner** = 분쟁 해결 책임자.
> **Code/UI 매핑**: 표준어 ↔ 코드 식별자 ↔ 화면 라벨을 일치시킨다(도메인이 "주문한다"면 코드도 `placeOrder`, `UserRequestsProduct` 금지). → 린트 타깃.
> **See also**(skos:related) = *관련* 개념 연결(혼동쌍과 다름 — 혼동쌍은 아래 §Flagged).

## Bounded contexts (같은 단어, 맥락별 다른 뜻)

> DDD: 한 단어가 컨텍스트마다 정당하게 다른 뜻(Fowler의 "meter" 3부서 사례; "Customer"@Sales ≠ @Support, "Order"@Checkout ≠ @Fulfilment). *false unification*(억지 단일 정의) 금지 — 같은 표준어를 컨텍스트 태그로 나눠 두 번 등재해도 된다. 단일 도메인이면 이 섹션 생략(암묵적 전역 컨텍스트).

| 용어 | 컨텍스트 A = 뜻 | 컨텍스트 B = 뜻 |
|---|---|---|
| (예: Order) | Checkout = 결제 전 장바구니 확정 | Fulfilment = 출고 지시 단위 |

## 약어 (Acronyms)

> 첫 사용 시 풀어 쓰고, 약어는 표준어의 altLabel로 등재. **AI 위험**: 기술 약어는 일반어에서 다의어라 모델이 엉뚱하게 해석 → 반드시 여기 고정.

| 약어 | 정식 명칭(표준어) |
|---|---|
| (예: SLA) | Service Level Agreement |

## Flagged ambiguities (혼동 주의 쌍)

> 자주 섞이는 개념쌍을 *구분 기준* 1문장으로 박제.

- **A vs B** — A는 …, B는 …. 절대 혼용하지 않는다.

## 예시 대화 (올바른 용어 사용 시연)

> 실제 역할 간 대화로, 위 용어가 *현장에서 어떻게 구분되어 쓰이는지* 시연한다. 정의보다 강력한 학습 장치.

> **역할1**: (잘못 섞은 질문)
> **역할2**: 아니요, 그건 **<표준어>**라 …. (올바른 구분 설명)

---
**AI 스티어링·강제**:
- **그라운딩 훅**: 이 파일을 `CLAUDE.md`/`AGENTS.md`/`llms.txt`에서 링크해 매 세션 컨텍스트에 주입(도메인 용어 주입 = 환각·드리프트 감소). 형식을 규칙적으로 유지(RAG·린트 파싱용).
- **린트 훅(`_Avoid_` = 조언 아닌 가드레일)**: 문서·PR·UI 카피·코드 식별자에 `_Avoid_` 용어가 나타나면 플래그(Vale류 규칙 / CI 체크) — canon을 문서에서 *강제*로.

**원칙**: 같은 개념엔 표준용어 1개(prefLabel). `_Avoid_`(hiddenLabel)로 버려진 표현을 박제해 산출물·코드·디자인 전반의 용어 일관성을 강제한다. 표준어는 코드·UI에 *그대로* 흐른다(유비쿼터스 언어). 맥락별 다른 뜻은 억지로 합치지 말고 컨텍스트로 나눈다. 혼동 쌍은 정의가 아니라 *예시 대화*로 시연하는 게 가장 잘 박힌다.
