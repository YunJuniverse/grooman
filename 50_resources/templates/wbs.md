**작성 규칙** (PMBOK Practice Standard for WBS):
- **1 Task = 0~5일** (초과 시 분해). 계층형 1.0 → 1.1 → 1.1.1. **최하위 = work package**(담당·기간·산출물·리스크·견적 단위).
- **100% 규칙**: 하위의 합 = 상위 범위의 100% — 모든 산출물을 담고 *범위 밖 작업은 넣지 않는다*(완전성·과잉 둘 다 방지).
- **PM·QA·검수도 산출물로 포함** — 흔히 누락돼 공수가 과소 산정됨(관리·품질 활동도 일이다).
- **산출물(명사) 중심, 활동(동사) 아님**: "설계"❌ → "정의서 v1.01"✅(deliverable-oriented, PMBOK).
- 일정은 업무시간 기준 **0.8 진수**(점심 제외), 전체에 **리스크 버퍼 0.3~0.5배**.
- **Task = 견적·일정 롤업 근거**: work package 값이 수익율표·수행계획으로 roll up(WBS=견적·통제 baseline). **비고 열 = mini WBS-dictionary**(1줄 설명·의존성·리스크).
- Task마다 **담당자 1명**. 작성은 합의 절차: 초안 → 직무별 수치 조정 → '팀의 버전'. 숨겨진 의존성·QA 일정 누락 주의.
