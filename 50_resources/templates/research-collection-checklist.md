### B. 프로젝트 셋업 discovery (수주·스코핑 입력)

> SOW/BANT 기준 — 이 4버킷이 수익율표·수행계획의 입력이 된다.

| 구분 | 수집자료 항목 | 적용(√) | 출처/수집일 | 담당 |
|------|------|------|------|------|
| 사업목표·성공기준 | 원하는 결과(outcome) · 성공 KPI · 인수기준 | | | |
| 예산·일정·권한(BANT) | 예산 범위 · 목표 일정 · **의사결정권자(서명 주체)** | | | |
| 제약·컴플라이언스 | 기술 제약 · 규제/보안(개인정보·SOC2/GDPR 해당 시) | | | |
| 기존자산·연동 | 현행 시스템·데이터·접근권한 등 **수행 의존 입력물** | | | |

**원칙**: A(시장) + B(프로젝트 셋업)를 빠짐없이 채운다. 적용(√)에서 빠진 항목은 *수집 불요*인지 *수집 누락*인지 담당자가 명시 — 빈칸 방치 금지. B는 예산·권한·기존자산이 빠지면 수익율·수행계획을 못 세운다.
